# Portafolio Hanna

Portafolio web con una portada interactiva y piezas individuales en paginas separadas.

## Estructura

- `index.html`: portada del proyecto.
- `script.js`: logica de la espiral y navegacion de la portada.
- `styles.css`: estilos globales de la portada.
- `001.html` a `005.html`: piezas individuales.
- `archivos-olvidados-media/`: librerias y medios usados por `002.html`.

## Convencion de cambios

- `main` queda para versiones estables.
- Cada cambio nuevo deberia salir de una rama corta: `fix/...`, `feat/...`, `style/...`, `chore/...`.
- Los commits deberian describir una sola intencion por cambio.

## Versiones

- No etiquetar cada commit.
- Usar tags solo para puntos importantes del proyecto.
- Mantener como referencia solo la version estable mas reciente, salvo que una entrega necesite conservar otra.

## Autoria

- No se fija autoria local del repositorio por ahora.
- Si despues se define una autoria explicita para commits futuros, el nombre a usar seria `Hanna Salazar`.
